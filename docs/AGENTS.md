# Documentation Agent Notes

Scope: planning, API/domain references, deployment notes, and task status.

- Do not load every doc. Start with `INDEX.md`, then open only the relevant
  domain file.
- `TASK_BOARD.md` is the task/status source of truth when a task ID is involved.
- `site/` contains large external provider reference dumps; open only the
  specific provider file needed for ingestion/API work.
- Keep docs synchronized with code only when the touched behavior, API contract,
  setup, or status changes.
- Avoid copying long code or command outputs into docs; link or summarize.
